# 파일 첨부 확정 스펙

## 기능 개요

노트에 파일을 추가하고 관리할 수 있다.

## 데이터 구조

### AttachedFile 타입

```ts
interface AttachedFile {
  id: string; // 삭제 시 식별용 고유 ID
  name: string; // 원본 파일명 (표시 및 다운로드 시 사용)
  type: string; // MIME type (이미지/텍스트 구분용)
  size: number; // 바이트 단위 크기
  data: string; // Base64 인코딩 데이터
}
```

### Note 타입 변경

```ts
interface Note {
  // 기존 필드 유지
  id: string;
  title: string;
  content: string;
  createdAt: string;
  updatedAt: string;
  // 추가 필드
  files: AttachedFile[]; // 첨부 파일 없으면 빈 배열 []
}
```

## 저장 방식

- Base64로 변환하여 `db.json`에 저장

## 제약 조건

| 항목                  | 값                        |
| --------------------- | ------------------------- |
| 파일당 최대 크기      | 2MB                       |
| 노트당 최대 첨부 개수 | 2개                       |
| 허용 이미지 확장자    | jpg, jpeg, png, gif, webp |
| 허용 텍스트 확장자    | txt, md                   |

## 기능 요구사항

- 노트에 파일을 추가할 수 있다.
- 노트에 추가된 파일을 삭제할 수 있다.
- 이미지 파일은 썸네일 형태로 파일 목록에 표시된다.
- 텍스트 파일은 파일 이름만 표시된다.
- 파일을 선택하면 PC에 다운로드할 수 있다.

## UI 배치

NoteEditor 하단에 파일 섹션 추가:

```
[제목 입력]
[내용 입력]
─────────────────
파일 첨부 버튼
[썸네일 or 아이콘] 파일명  [삭제]
[썸네일 or 아이콘] 파일명  [삭제]
```

## 유효성 검사 및 에러 처리

- 2MB 초과 파일 첨부 시 에러 메시지 표시 후 차단
- 노트당 2개 초과 첨부 시도 시 에러 메시지 표시 후 차단
- 허용되지 않은 파일 타입 첨부 시 에러 메시지 표시 후 차단
- 동일 파일명 중복 첨부 시 에러 메시지 표시 후 차단

## 노트 삭제 시 동작

- 노트 삭제 시 첨부 파일도 함께 삭제된다.
- 별도 경고 없이 즉시 삭제한다.
